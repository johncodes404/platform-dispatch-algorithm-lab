# 平台算法权力实验室

一个可以亲手改变派单权重、观察劳动风险如何重新分配的互动学习项目。

线上体验将通过 GitHub Pages 发布。网站使用纯 HTML、CSS 和 JavaScript，不需要登录，也不会上传或保存你的操作数据。

## 可以怎么玩

- **派单实验**：拖动效率、超时、成本、疲劳与公平权重，观察三名骑手的得分和获单结果；
- **事件卡**：加入暴雨、商家晚出餐、电梯故障，观察预测误差由谁承担；
- **疲劳保护线**：比较“把安全设成软指标”和“把安全设成硬约束”的差别；
- **午高峰模拟**：连续运行六轮，观察疲劳、收入差距与履约率如何累积；
- **制度对照**：把效率优先和劳动者友好两套目标函数放在同一笔订单上比较；
- **原理拆解**：从管理层目标到系统执行，理解算法是怎样被组织写出来的。

## 本地运行网页

网页是静态文件，可以直接打开：

```text
public/lab/index.html
```

也可以在仓库根目录启动任意静态服务器：

```bash
python -m http.server 8000 --directory public/lab
```

然后访问 `http://localhost:8000`。

## Python 教学模型

`python/` 目录保留了对应的 Python 派单模型和自动测试。它适合逐行阅读算法怎样完成预测、归一化、加权和选择。

```bash
cd python
python dispatch_simulator.py
python -m unittest -v
```

## 核心观点

算法不公平不一定源于“代码写错了”。更多时候，是企业把准时率、履约成本和订单量写进目标函数，却没有给疲劳、等餐风险与收入稳定同等位置。

> 算法治理不是让公众阅读几百万行源码，而是让目标、约束、数据、结果和责任能够被追问。
