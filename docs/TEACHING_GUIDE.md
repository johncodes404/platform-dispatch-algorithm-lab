# 课堂使用指南

## 1. 适用主题

算法素养、人工智能与社会、劳动教育、经济学、信息技术、公共管理、社会学、伦理与法治。

本项目不是让学生记住一套“正确权重”，而是练习观察、预测、比较、举证和制度设计。

## 2. 课前准备

教师提前：

1. 打开在线页面并完成一次完整操作；
2. 阅读 README 和 GAME_DESIGN；
3. 选择适合年龄的指标和角色；
4. 决定是否讨论真实案例，并核查来源；
5. 准备无法联网时的截图或纸面备选；
6. 提醒学生角色观点不等于本人观点。

学生不需要编程基础。

## 3. 45 分钟推荐流程

| 时间 | 活动 | 教师任务 |
| --- | --- | --- |
| 0–5 分钟 | 提问：最快派单一定最好吗 | 收集直觉，不立即判断 |
| 5–10 分钟 | 阅读三位骑手信息并预测 | 追问依据和遗漏数据 |
| 10–18 分钟 | 运行效率优先政策 | 对照分项得分与解释 |
| 18–25 分钟 | 调整为安全/公平并触发事件 | 比较谁受益、谁承担代价 |
| 25–33 分钟 | 运行高峰连续模拟 | 观察单次选择如何累积 |
| 33–41 分钟 | 小组设计一条治理规则 | 要求规则可执行、可监督 |
| 41–45 分钟 | 分享与退出卡 | 总结证据和未解决问题 |

## 4. 三类核心问题

### 观察

- 输入数据有哪些？哪些可能测不准？
- 谁被硬约束拦截？谁只是得分较高？
- 改变权重后，哪一项分数最影响结果？
- 单轮和六轮结果有什么不同？

### 解释

- 同样的数据为什么会产生不同决定？
- 商家、电梯或天气造成的延误最后算在谁身上？
- 平均 ETA 是否足以说明劳动风险？
- 把安全设成权重和设成红线有什么差别？

### 设计

- 哪些规则应公开，公开到什么程度？
- 谁可以修改权重，劳动者如何参与？
- 预测错误如何申诉、举证和补偿？
- 新增一项指标会带来哪些新的操纵空间？

## 5. 年龄调整

### 小学高年级

只用效率、安全、公平三个词；不讲公式，使用“规则卡”。让学生用一句话说明每个选择对四个角色的影响。

### 初中与高中

展示归一化和权重。让学生先预测，再计算或比较。引入平均数、最大值、外部性和反馈循环。

### 大学

要求阅读 Python 模型，批判数据代理、目标函数和候选集合效应。可提交算法影响评估、政策备忘录或模型修改 PR。

## 6. 学习评价

建议使用四项简短量规，每项 0–2 分：

| 能力 | 0 | 1 | 2 |
| --- | --- | --- | --- |
| 因果追踪 | 只描述结果 | 能指出一个参数 | 能连接输入、规则、结果与反馈 |
| 多角色理解 | 只站一个立场 | 能说出另一方利益 | 能解释至少两方约束与冲突 |
| 证据使用 | 只有观点 | 引用一个页面指标 | 用多项指标支持并说明局限 |
| 治理设计 | 口号式建议 | 有具体规则 | 规则可执行、监督、申诉和修正 |

## 7. 退出卡

请学生用三句话完成：

1. 我改变了 ______，结果 ______。
2. 这个结果对 ______ 有利，但把 ______ 转移给了 ______。
3. 如果我是规则制定者，我会加入 ______，并用 ______ 检查它是否有效。

## 8. 教师注意事项

- 不把某个预设称为唯一正确答案；
- 避免把高疲劳个人归因为“不自律”；
- 区分模拟参数、研究证据和个人经验；
- 真实劳动经历可以自愿分享，不追问身份细节；
- 讨论事故和收入压力时给学生退出敏感话题的空间；
- 如果学生发现模型不合理，把它当作学习成果而不是操作错误。

欢迎通过课堂反馈 Issue 分享：年龄、时长、最有启发的时刻、最困惑的地方和建议。不要提交学生姓名或可识别信息。
