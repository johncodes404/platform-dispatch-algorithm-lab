# 对外介绍与参与招募

## 项目一句话

平台算法权力实验室：亲手调整派单规则，看见效率、利润、安全与公平如何改变劳动风险和收益分配。

## 朋友圈短版

我做了一个开源互动小项目「平台算法权力实验室」。

你可以像平台规则制定者一样调整派单权重，观察同一笔订单为什么会派给不同骑手，以及效率、成本、安全和公平之间的取舍。它不是在复刻某家平台的私有算法，而是一个透明、可质疑的教学模型。

我希望下一步把它做成多角色课堂游戏，让大家分别站在投资者、平台、劳动者、商户、消费者和监管者的位置思考：谁分蛋糕，谁担风险，谁来定规则？

欢迎教师、学生、骑手、研究者、设计师和开发者来试玩、提建议、一起共创。

在线体验：https://johncodes404.github.io/platform-dispatch-algorithm-lab/
项目仓库：https://github.com/johncodes404/platform-dispatch-algorithm-lab

## 朋友圈故事版

算法为什么会让人觉得“不公平”？

很多时候，代码只是最后一层。真正影响结果的，还有谁定义效率、哪些风险被计入、哪些被忽略、什么是可以交换的成本、什么必须成为不可突破的红线。

我把这些问题做成了一个可以玩的开源网页：「平台算法权力实验室」。你可以调整效率、成本、疲劳和公平等权重，触发暴雨、商家延误、电梯故障，再观察派单结果和六轮高峰之后的劳动后果。

这不是任何真实平台的源码，而是一台透明的“思考机器”。

接下来我想把它发展成课堂小游戏：同学们分别扮演投资者、平台、劳动者、商户、消费者和政府，在有限资源和突发事件里讨论利润、责任、风险与蛋糕如何分配。它可以服务大中小学的算法素养、劳动教育、经济与社会课程。

项目已经开源。无论你会不会写代码，都欢迎参与：试玩、找问题、贡献课堂案例、补研究资料、设计角色与事件，或者帮忙做无障碍和视觉体验。

在线体验：https://johncodes404.github.io/platform-dispatch-algorithm-lab/
参与共创：https://github.com/johncodes404/platform-dispatch-algorithm-lab/issues/new/choose

## 面向不同参与者的邀请

- 教师：试用课堂流程，告诉我们哪个环节最能引发讨论；
- 学生：指出看不懂、不好玩或不公平的地方；
- 劳动者：在保护隐私前提下纠正不真实的设定；
- 研究者：补充资料、模型局限和概念边界；
- 设计师：改善信息呈现、角色体验与无障碍；
- 开发者：补测试、交互、模拟与部署；
- 家长和公众：提出你认为课堂最该讨论的问题。

## 发布前检查

- 在线链接可以正常打开；
- 配一张清晰截图或短录屏；
- 说明项目是教学模型，不是平台真实源码；
- 邀请具体类型的反馈，而不只是“欢迎点赞”；
- 不公开参与者、学生或劳动者的个人信息；
- 把重要反馈整理为 Issue，避免只留在朋友圈评论里。
