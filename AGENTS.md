# AGENTS.md

本文件是给 AI 编程助手和自动化开发工具的项目级指令。开始修改前，先阅读 README.md、docs/DEVELOPMENT.md、docs/STRUCTURE.md 与本文件。

## 1. 项目使命

平台算法权力实验室通过透明、可操作的模型，帮助学习者理解平台目标、派单规则、劳动风险、收益分配和公共治理之间的关系。

优先级从高到低：

1. 教学含义准确、可解释；
2. 学习者能够观察“参数—决策—后果”的因果链；
3. 对劳动者与不同社会角色保持尊重；
4. 易访问、易运行、易协作；
5. 最后才是更复杂的功能和视觉效果。

## 2. 事实与表达边界

- 这是简化的教学模拟，不是真实平台源码，也不能暗示复刻了某家公司的私有算法。
- 不把任何角色写成脸谱化的“好人”或“坏人”。角色行为应来自目标、资源、信息和约束。
- 不使用真实劳动者的个人数据、轨迹或身份信息。
- 数值是用于比较的教学参数；如声称来自研究或现实数据，必须给出可核验来源。
- 面向未成年人时，不使用羞辱、恐吓、赌博式付费或成瘾机制。
- 任何新增功能都应帮助学习者提出问题，而不是替他们给出唯一价值判断。

## 3. 核心技术约定

- 网站保持纯静态，入口为 public/lab/index.html。
- 浏览器状态和交互逻辑集中在 public/lab/app.js，样式在 public/lab/styles.css。
- Python 模型用于教学、验证和未来批量实验，不是网页运行依赖。
- 所有评分指标先归一化到 0–1；综合成本越低，派单优先级越高。
- 安全、休息、容量等硬约束必须先于权重评分执行。
- UI 中的解释必须由实际计算结果生成，不得出现与得分不一致的硬编码结论。
- 修改核心公式、权重含义或情境效果时，同时检查网页模型、Python 模型、测试和文档；如果二者有意不同，必须写明原因。
- 不引入后端、账号、数据库、分析追踪或外部 SDK，除非 Issue 已明确讨论其教学必要性、隐私方案和维护成本。

## 4. 修改流程

1. 明确本次修改解决的学习问题和受影响文件。
2. 阅读相关实现和现有测试，不重复创建已有能力。
3. 采用最小、可审查的改动；不要顺手重构无关区域。
4. 为不直观的业务规则写“为什么”的注释，避免逐行翻译代码。
5. 更新相应文档和测试。
6. 完成验证清单，并在提交或 PR 中记录结果。

## 5. 代码风格

### JavaScript

- 使用 const/let，不使用 var。
- 函数保持单一职责；计算函数尽量无副作用。
- 业务概念使用稳定的英文标识符，界面文案使用自然、易懂的中文。
- 新增核心函数时补充 JSDoc，说明输入、输出、单位和边界。
- DOM 查询集中管理，避免在循环中反复查找相同节点。
- 动画需尊重 prefers-reduced-motion；交互不能只依赖颜色表达。

### Python

- 使用类型标注、dataclass 和清晰的 docstring。
- 算法计算与命令行演示分离。
- 公共函数应说明参数单位、返回值和异常条件。
- 新增规则时覆盖正常、边界和不可行候选三类测试。

### HTML/CSS

- 优先使用语义化标签、原生按钮与表单控件。
- 表单必须有 label，键盘操作和焦点状态必须可见。
- 保持移动端可用；不得只为桌面宽屏设计。
- 复用现有 CSS 变量与组件类，避免无来源的魔法数。

## 6. 验证清单

提交前至少执行：

    python -m unittest python/test_dispatch_simulator.py
    python -m http.server 8000 --directory public/lab

手动检查：

- 页面首次加载无控制台错误；
- 四组权重、预设和情境可以切换；
- 派单结果、排名和解释同步更新；
- 连续模拟可以启动、暂停和重置；
- 375px 左右窄屏可读；
- 仅使用键盘可以完成主要操作；
- 新文案没有把模拟结果描述成真实世界事实。

## 7. 文件放置规则

- 可运行网页：public/lab/
- Python 教学模型：python/
- 长篇说明：docs/
- 协作模板和部署：.github/
- 面向 AI 的全局约束：AGENTS.md
- 面向人的项目入口：README.md

不要把构建产物、编辑器缓存、系统文件或真实个人数据提交到仓库。

## 8. AI 交付格式

AI 完成任务时，应说明：

- 改了什么以及它服务的学习目标；
- 涉及哪些文件；
- 执行了哪些验证；
- 仍存在的假设、风险或后续工作。

如需求会改变教育立场、数据来源、隐私边界或总体架构，先在 Issue/讨论中提出选项，不自行替项目作重大决定。
